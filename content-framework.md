# 多身份分支叙事副本 · 内容框架

---

## 一、框架总览

本框架用于批量生产"多身份 × 多角色 × 线性路径结局"的副本。每个副本讲述一个完整故事，玩家选择女主身份和男主身份进入，通过收集钥匙完成任务、离开副本。不同身份组合产生不同的故事体验，但共享同一条主线脊柱。

### 核心公式

```
1个副本 = 1条主线脊柱（固定大节点序列）
        × 3个女主身份（决定任务展开方式）
        × 3个男主身份（决定配合/冲突方式）
        × N个选择点（决定结局走向）
        × M枚钥匙（分布在主线和探索中）
```

### 五层结构

```
第一层 · 主线脊柱        固定的大节点序列，讲述完整故事
第二层 · 身份展开        女主身份决定每个节点的具体危机和解法
第三层 · 任务与男主配合   节点内的小任务链，男主按身份参与/反应
第四层 · 情感与选择       选择点 + 情感节点，塑造关系走向
第五层 · 结局收束        选择点组合 → 线性映射到结局
```

---

## 二、第一层 · 主线脊柱

### 定义

主线脊柱是副本的故事骨架——一系列**固定的大节点**，所有身份组合都会经过，顺序不变。节点之间有因果关系（A引发B，B暴露C，C逼出D）。

### 设计要求

| 项目 | 要求 |
|------|------|
| 节点数量 | 建议 3-5 个大节点，对应三幕结构（入局 / 探局 / 破局）|
| 因果链 | 每个节点必须由上一个节点的结果触发，不能是独立事件并列 |
| 暗线 | 必须有一条贯穿全局的暗线（幕后势力/阴谋），在节点间逐步揭露 |
| 转折 | 每个大节点结束后应有转折——暴露新信息，改变局势，推动进入下一节点 |
| 终局压力 | 最后一个节点必须是不可逆的高压事件（围剿/崩塌/对决），逼出所有角色的最终立场 |

### 节点模板

```
大节点 X · [名称]
├─ 节点目标：玩家在此节点要完成什么
├─ 暗线推进：幕后势力在此节点做了什么
├─ 转折：节点结束后揭露的新信息
└─ 通向：下一个节点的触发条件
```

### 三幕节奏建议

```
第一幕 · 入局（1-2个大节点）
  - 女主进入副本世界，解决即时危机，站稳脚跟
  - 摸清世界规则和权力结构
  - 幕末获得关键世界观信息，引出第二幕事件

第二幕 · 探局（1-2个大节点）
  - 参与/应对副本世界的重大事件
  - 暗线从幕后走到台前，男主秘密暴露
  - 情感关系在协作/冲突中深化
  - 幕末真相大白，终局事件迫在眉睫

第三幕 · 破局（1个大节点）
  - 终局事件爆发，不可逆
  - 男主最终立场，受前两幕选择影响
  - 最后一枚钥匙获取
  - 结局分流
```

## 三、第二层 · 身份展开

### 定义

女主的3种身份决定了她在每个大节点中**用什么方式**完成目标。目标相同，手段不同。

### 设计要求

| 项目 | 要求 |
|------|------|
| 身份差异化 | 每个身份必须有**专属的玩法机制**（如观察系统/伪装系统/经济系统），贯穿全副本 |
| 同目标异手段 | 同一个任务目标，3种身份用完全不同的方式完成 |
| 第一幕危机差异 | 第一幕的身份危机允许3条线讲述**差异较大的故事**，因为此时尚未合流 |
| 后续节点收敛 | 第二幕起，3条线的展开方式不同但**故事走向收敛**（同一个转折、同一个暗线揭露） |

### 身份模板

```
身份 X · [名称]
├─ 一句话定位：她是谁，擅长什么
├─ 专属玩法：贯穿全副本的操作机制（如鹰眼/伪装/买通）
├─ 玩法首秀：在第一幕哪个任务中首次引入
├─ 第一幕危机：只属于这个身份的即时危机（可与其他身份差异大）
├─ 对每个男主的前史：进入副本前与男主的关系基础（陌生/听闻/有渊源）
└─ 钥匙探索逻辑：这个身份在任务流程中会自然经过哪些场所（决定探索型钥匙的放置）
```

### 玩法机制设计规则

- 玩法机制必须**反复使用**（至少在3个不同任务中出场），不能是一次性噱头
- 每次使用应有**递进**（第一次简单教学，后续逐渐复杂）
- 玩法机制在不同节点中的表现应有变化（例：观察系统在赌局中读微表情，在危机中追踪线索，在战斗中发现机关）

## 四、第三层 · 任务与男主配合

### 任务节点的两种版本

每个大节点内包含若干小任务。任务分两种版本：

| 版本 | 含义 | 适用场景 |
|------|------|---------|
| **固定版** | 每个女主身份只有一种做法，无选择分支 | 用于铺垫/过渡/教学类任务 |
| **选择版** | 每个女主身份有2个选择（A/B），走向不同 | 用于关键任务，选择会影响后续情感/结局 |

### 任务密度控制

> **核心节奏：固定与选择交替出现，避免连续选择疲劳**

建议每个大节点内：
- 2个固定版任务 + 2个选择版任务（总共约4个小任务）
- 或 3个固定版 + 1个选择版（轻量节点）
- **不要每个任务都是选择版**——玩家选择疲劳会稀释关键选择的分量

### 男主配合模式

每个任务中，男主按其身份以不同方式参与。参与模式分3种：

| 模式 | 描述 | 举例 |
|------|------|------|
| **辅助** | 男主主动帮女主，提供信息/资源/掩护 | 提供线索、安排机会、武力护送 |
| **被动卷入** | 男主因身份/职责出现在场景中，产生交集但非刻意 | 巡逻时撞见、在牌桌上被观察 |
| **冲突** | 男主的利益/立场与女主产生摩擦 | 阻挡去路、质疑行为、争夺资源 |

### 男主反应矩阵

每个选择版任务需要填写一张反应矩阵：

```
任务名 · 女主身份 X
┌──────────┬──────────┬──────────┬──────────┐
│ 女主选择  │  男主甲   │  男主乙   │  男主丙   │
├──────────┼──────────┼──────────┼──────────┤
│ 选择 A   │ 反应描述  │ 反应描述  │ 反应描述  │
│ 选择 B   │ 反应描述  │ 反应描述  │ 反应描述  │
└──────────┴──────────┴──────────┴──────────┘
```

### 设计要求

- 男主的反应必须**符合其身份逻辑**——他为什么会这样做，基于他的身份/秘密/目标
- 同一个男主在不同女主身份线中的反应可以不同，但**人设必须一致**（不能荷官在赌神线是冷酷的在千金线是温柔的，除非有合理的身份逻辑）
- 选择A和选择B不应是"好/坏"的二元对立，而是**不同的做事方式**，各有合理性

## 五、第四层 · 情感与选择

### 情感节点

情感节点是男女主关系发展的场景，嵌入在任务之间或之后。

#### 两种情感节点

| 类型 | 描述 | 内含选择 |
|------|------|---------|
| **大情感节点** | 受前置任务选择影响，呈现不同场景变体；内含情感选择 | 有，且是选择点（决定结局） |
| **小情感节点** | 固定场景，展示男主的性格/处境/秘密；无选择或仅有风味选择 | 无，或仅做铺垫 |

#### 情感节点的级联设计

```
任务选择（A/B）
  ↓ 影响
大情感节点（呈现A版本场景 或 B版本场景）
  ↓ 内含
情感选择（甲/乙）← 这是选择点，决定结局
```

**关键设计要求：**
- 先前的任务选择决定**情感节点呈现哪个版本的场景**（例：偷了工具 → 两人有共同秘密 → 情感场景围绕秘密展开；没偷 → 没有秘密 → 情感场景围绕实力展开）
- 在不同版本的场景中，情感选择的**选项措辞可以不同**，但**最终对结局的作用等价**（甲路径的两个选项和乙路径的两个选项映射到同一个选择点变量）

### 选择点

选择点是影响结局走向的关键选择。

#### 设计原则

| 原则 | 说明 |
|------|------|
| **非二元对立** | 选择不是"对他好 vs 对他差"，而是"不同的相处方式"。两个选项都是合理的行为，只是方向不同 |
| **数量控制** | 建议 3 个选择点。3×2=8 条路径，收束为 4-5 种结局——足够丰富又不至于失控 |
| **均匀分布** | 选择点应分布在不同幕（第一幕末 / 第二幕中 / 第二幕末或第三幕前），不要扎堆 |
| **逐步升级** | 第一个选择点关于"我们是什么样的关系"，第二个关于"危机中我怎么对待你"，第三个关于"最终时刻我选择什么" |

#### 选择点模板

```
选择点 X · 位于 [节点位置]
├─ 场景背景：发生了什么导致需要选择
├─ 男主甲线：
│   ├─ 选项A [措辞]：男主反应 + 关系走向
│   └─ 选项B [措辞]：男主反应 + 关系走向
├─ 男主乙线：（同上结构）
└─ 男主丙线：（同上结构）
```

## 六、第五层 · 结局收束

### 线性路径映射

结局由选择点的**具体组合**直接决定——不做累计计数、好感度累加或模式匹配。

```
选择点① × 选择点② × 选择点③ = 具体路径 → 对应结局
```

### 结局类型

建议每个副本设 4 种基础结局 + 1 种特殊变体：

| 结局 | 定位 | 触发逻辑 |
|------|------|---------|
| **同行** | 最好结局，二人一起离开 | 需要多次选择都偏向"走近对方" |
| **独行** | 中性结局，完成任务各走各路 | 选择偏向"保持距离但务实" |
| **对峙** | 激烈结局，最后为钥匙/立场对抗 | 选择模式让对方"看不清你" |
| **牺牲** | 悲剧结局，男主为信念/保护而死 | 选择让他"被触动但无法留在身边" |
| **困局** | 失败结局，钥匙未集齐无法离开 | 前置条件：钥匙不全 |

### 特殊结局变体

对峙结局末尾可增设**额外一选**：玩家在胜利后选择是否将钥匙让给对方——
- 带走 → 对峙收场（正常完成）
- 留给他 → 困局变体（任务失败，但尾声展示男主念念不忘的画面）

### 路径映射表模板

每条男主线一张表：

```
| ① | ② | ③ | 结局 | 弧线说明 |
|---|---|---|------|---------|
| A | A | A | 同行 | [为什么这组合导向同行] |
| A | A | B | 牺牲 | [为什么这组合导向牺牲] |
| ... | ... | ... | ... | ... |
```

### 设计要求

- 每种结局至少有 1 条路径可达，不能有结局是"设计了但没有路径到达"
- 每条路径的弧线说明必须**叙事自洽**——玩家回顾选择序列时应该觉得"这个结果合理"
- 不同男主线的结局分布可以不均匀（例：某个男主的"同行"特别难达成，只有1条路径——这是人设决定的）
- 3个选择点的措辞不需要跨男主线统一——每条男主线的选择是**为他量身设计的**，围绕他的核心矛盾展开

## 七、钥匙系统

### 核心设定

- 玩家**进入副本前已知**钥匙在这个时代的形态（如金筹码/玉佩/信物……）
- 钥匙是特殊物品，不在日常流通中，副本世界中仅少数人持有或知道其存在
- 集齐所有钥匙 = 离开副本的条件。缺任何一枚 → 困局（失败结局）

### 钥匙分布规则

| 规则 | 说明 |
|------|------|
| **总数固定** | 建议 5 枚。太少缺乏收集感，太多难以合理安排 |
| **两种获取方式** | 主线获取（任务奖励/交易/赢取）+ 探索获取（在场景中发现） |
| **比例建议** | 3枚主线 + 2枚探索，或 2+3。不要全是主线（没有探索乐趣）也不要全是探索（缺乏任务驱动） |
| **节奏分布** | 每幕至少 1 枚。不要前半段拿完后半段无事可做 |

### 主线钥匙设计要求

| 要求 | 说明 |
|------|------|
| **获取必须与任务绑定** | 钥匙获取是任务的直接目标或副产品 |
| **信息前置** | 玩家在获取前必须**已经知道这枚钥匙的存在和位置**（通过NPC提及/世界观铺垫/线索拼合），不能凭空冒出 |
| **NPC持有理由** | 如果钥匙在NPC手中，必须交代**为什么他有这个东西**（历史渊源/职位权限/偶然获得），不能是"他就是有" |
| **不同身份不同手段** | 3种女主身份用不同方式从同一来源获取（赌赢/威胁/收购……），但来源相同 |

### 探索钥匙设计要求

| 要求 | 说明 |
|------|------|
| **藏在必经之路上** | 钥匙所在的场所必须是某个任务的必经路线，不能藏在完全与主线无关的角落 |
| **身份决定路线** | 不同身份经过的场所不同 → 不同身份在不同地方发现同一枚钥匙 |
| **被动发现而非主动搜索** | 玩家在执行任务的过程中"顺路看到"，而不是专门去找。因为玩家已知钥匙形态，看到时自然会拿取 |
| **合理存在** | 钥匙出现在某处必须有世界观解释（旧柜子里的遗物/扒手偷来的杂物/老人的信物），不能是"桌上放着一个" |

### 钥匙失败节点

至少设 1 个"可永久丢失"的钥匙失败节点——

- 探索钥匙允许"跳过稍后回来取"，但特定失败条件（如场景坍塌）导致永久丢失
- 这给玩家制造紧迫感："先拿钥匙还是先处理危机"
- 永久丢失 → 直接进入困局结局线，不可挽回

### 钥匙信息流设计

钥匙的信息应该像拼图一样逐步揭露：

```
第一幕初 → 玩家已知：钥匙总数 + 形态
第一幕中 → 发现第1枚（探索，确认形态正确）
第一幕末 → 得知世界观信息（为什么这种物品存在，谁可能有）→ 引出第2枚来源
第二幕   → 信息进一步聚合 → 得知第3枚位置 + 发现第4枚（探索）
第三幕   → 推断出最后1枚位置（此前只知道"大概在某处"）
```

## 八、空间与场景

### 公共空间

副本需要设计一组**公共空间**，所有身份组合都可以进入：

- 建议 4-6 个功能区域（前台/后台/VIP区/底层/仓库/管理层……）
- 每个区域有明确的**功能定位**和**可交互要素**
- 不同身份在不同区域的**活动权限不同**（某些身份需要伪装/买通才能进入特定区域）
- 钥匙（探索型）应分布在不同区域，且与身份的活动路线绑定

### 男主专属空间

每个男主身份解锁 1 个**独占场景**——其他男主线无法进入：

| 要求 | 说明 |
|------|------|
| **绑定男主身份** | 场景反映男主的内心世界/秘密/私人空间 |
| **剧情功能** | 必须绑定至少1个主线任务节点（不是纯观赏） |
| **情感节点承载** | 关键情感选择（选择点）应尽可能发生在专属场景中，增强沉浸感 |
| **时间线绑定** | 专属场景在第二幕中首次开放（第一幕太早关系不够，第三幕太晚来不及） |

## 九、暗线与NPC

### 暗线设计

每个副本必须有一条暗线——一个幕后势力/阴谋在故事推进中逐步揭露。

#### 暗线与男主的关联

**每个男主身份都必须与暗线有关联**——这是制造"男主秘密暴露"戏剧高潮的来源。

```
男主甲 · 与暗线的关联：[关联方式 + 暴露后果]
男主乙 · 与暗线的关联：[关联方式 + 暴露后果]
男主丙 · 与暗线的关联：[关联方式 + 暴露后果]
```

关联类型：
- **被胁迫**：男主被暗线势力要挟做内应（被迫，抗拒但无奈）
- **是目标**：男主本身就是暗线势力攻击的目标
- **是棋子**：男主被利用但不自知/被债务/人质要挟做具体的事

#### 暗线揭露节奏

```
第一幕 → 异常信号（玩家感觉"有什么不对"但不知道是什么）
第二幕前半 → 异常汇聚（多个线索指向同一方向）
第二幕中 → 暗线浮出水面（男主秘密暴露——他与暗线的关联被揭穿）
第二幕末 → 真相完整揭露（幕后计划全貌清晰）
第三幕 → 暗线势力行动（终局事件 = 暗线计划的执行/高潮）
```

### 功能NPC

除男主外，副本需要若干功能NPC：

| NPC类型 | 作用 | 数量建议 |
|---------|------|---------|
| **钥匙持有者** | 持有主线钥匙，需要通过任务获取 | 1-2个 |
| **信息提供者** | 提供世界观/暗线/钥匙线索 | 2-3个 |
| **障碍制造者** | 阻挡女主或制造危机 | 1-2个 |
| **暗线关联者** | 与暗线势力相关的人（幕后黑手/帮凶/受害者/老人） | 1-2个 |

**NPC设计要求：**
- 每个NPC必须有**存在的理由**——他在这个世界里是谁，为什么在这里
- 钥匙持有者必须有**持有理由**和**放手动机**
- NPC不能只为一个身份线服务——他在3条女主线中都应有角色（哪怕其中2条线只是路过/提及）

## 十、内容量估算

### 单副本内容矩阵

```
大节点数 × 每节点任务数 × (3女主身份 × 固定/选择版本) × 3男主反应 = 总内容单元
```

以"5大节点 × 平均4任务 × 3身份 × 2版本（平均）× 3男主"估算：
- 任务场景文本：~360 个内容单元
- 情感节点：~15 个（每幕1大 + 若干小）
- 结局文本：4种 × 3男主 = 12 段 + 特殊变体
- 总计约 400 个内容单元

### 可复用模式

以下模式可跨副本复用，降低单副本创作成本：

| 模式 | 说明 |
|------|------|
| **同目标异手段** | 每个任务 = 1目标 + 3女主方法 + 3男主变体 = 填7个单元 |
| **反应式插入** | 大场景共享核心文本 + [女主反应]×3 + [男主反应]×3 + [CP时刻]×9 |
| **身份风味** | 通用互动结构不变，身份只改表达方式（语气/行为/画面） |
| **路径映射** | 结局结构固定（3选择点×2选=8路径→4结局），换男主线时只需重填映射表 |

## 十一、铺量检查清单

创建新副本时按此清单逐项检查：

### 世界观层

- [ ] 副本世界设定完成（时代/地点/核心冲突）
- [ ] 暗线设定完成（幕后势力/目的/三步计划）
- [ ] 暗线与3个男主的关联各不相同且合理
- [ ] 公共空间 4-6 个，功能定位清晰
- [ ] 男主专属空间 3 个，绑定任务节点

### 主线层

- [ ] 大节点 3-5 个，因果链清晰，每个由上一个触发
- [ ] 每个节点有转折，推动进入下一节点
- [ ] 节点序列覆盖三幕结构

### 身份层

- [ ] 女主 3 种身份定位差异化，各有专属玩法
- [ ] 玩法机制至少在 3 个任务中出场
- [ ] 男主 3 种身份定位差异化，各有秘密/目标/帮女主的逻辑
- [ ] 男主 × 女主 前史 9 组都已设定

### 任务层

- [ ] 每个大节点内有固定版 + 选择版任务，密度合理
- [ ] 选择版任务的 A/B 非二元对立，各有合理性
- [ ] 男主反应矩阵已填写（每个选择版任务 × 3男主）
- [ ] 男主反应符合身份逻辑，跨女主线人设一致

### 情感层

- [ ] 大情感节点受前置任务选择影响（有场景变体）
- [ ] 情感节点内的选择措辞非二元对立
- [ ] 3 个选择点分布在不同幕，逐步升级
- [ ] 小情感节点铺垫充分

### 钥匙层

- [ ] 总数 5 枚，主线 3 + 探索 2（或 2+3）
- [ ] 每枚有信息前置（玩家获取前已知其存在）
- [ ] 探索钥匙在必经之路上，不同身份不同场所
- [ ] 至少 1 个可永久丢失的失败节点
- [ ] NPC持有理由已交代
- [ ] 钥匙形态在进入副本前已告知玩家

### 结局层

- [ ] 路径映射表 3 张（每条男主线）已填完
- [ ] 每种结局至少 1 条路径可达
- [ ] 每条路径弧线叙事自洽
- [ ] 特殊变体（如"念念不忘"）已设计
- [ ] 困局结局条件清晰（哪些失败点导致）

### 整体验证

- [ ] 选 1 个完整组合，从入局到结局走完——叙事连贯
- [ ] 选 1 个大节点，横向走 9 种组合——差异化且共享核心
- [ ] 因果链检查：每一步的触发条件合理
- [ ] 钥匙线检查：获取时机/引导方式/失败节点清晰
